NAME
    sha512sum - calculate or check SHA-512 fingerprints for files

WHAT IT DOES
    sha512sum creates a 128-character SHA-512 fingerprint (checksum) for
    each file. SHA-512 is one of the most powerful hash algorithms in the
    SHA-2 family. It produces very long fingerprints that are extremely
    hard to fake, making it excellent for security-sensitive verification.

HOW TO USE IT
    sha512sum [options] [FILE]...

    When no FILE is given, sha512sum reads from standard input.

OPTIONS
    -b, --binary
        Read the file in binary mode (marks output with *).

    -c, --check
        Read fingerprints from FILE and check that the files still match.

    --tag
        Print results in BSD style: SHA512 (filename) = fingerprint.

    --quiet
        When checking, only report files that FAILED.

    --status
        When checking, do not print anything; just use the exit code.

    --ignore-missing
        When checking, skip files that are listed but do not exist.

    --strict
        When checking, fail if any checksum lines are badly formatted.

    -w, --warn
        When checking, warn about badly formatted checksum lines.

    --help
        Show a short help message and quit.

EXAMPLES
    sha512sum bigfile.iso
        Print the SHA-512 fingerprint for a disk image.

    sha512sum *.pkg > checksums.txt
        Save fingerprints for all package files.

    sha512sum -c checksums.txt
        Check that all files still match their saved fingerprints.

SEE ALSO
    sha256sum(1), sha384sum(1), md5sum(1)
