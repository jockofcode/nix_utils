NAME
    cksum - create a checksum fingerprint for a file

WHAT IT DOES
    A checksum is like a fingerprint for a file. cksum reads a file and
    calculates a number that is unique to that file's contents. If you
    change even one letter in the file, the number will be different.

    This is useful for checking that a file was not changed or damaged
    when you copied or downloaded it.

HOW TO USE IT
    cksum [options] [FILE]...

    When no FILE is given, cksum reads from standard input.

OPTIONS
    -a TYPE, --algorithm=TYPE
        Choose which formula to use to calculate the fingerprint.
        Options: crc (default), bsd, sysv, crc32b, md5, sha1, sha256.

    --base64
        Show the fingerprint using base64 letters instead of a long number.

    -l BITS, --length=BITS
        For algorithms that support it, how many bits long the output
        fingerprint should be.

    --tag
        Print the result in BSD style: ALGO (filename) = fingerprint.

    -c, --check
        Instead of calculating, read a list of fingerprints from FILE
        and check that the files still match.

    --help
        Show a short help message and quit.

EXAMPLES
    cksum report.pdf
        Print the CRC checksum and size of report.pdf.

    cksum -a md5 photo.jpg
        Print an MD5 fingerprint for photo.jpg.

    cksum -a sha256 --tag installer.zip
        Print a SHA-256 fingerprint in BSD tag style.
