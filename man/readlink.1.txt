NAME
    readlink - show where a symbolic link points

WHAT IT DOES
    A symbolic link (symlink) is like a shortcut that points to another
    file or folder. readlink shows you the target — where the shortcut
    actually points to.

    With extra options, readlink can also resolve the full, real path of
    a file even if it goes through multiple levels of symlinks.

HOW TO USE IT
    readlink [options] FILE...

OPTIONS
    -f, --canonicalize
        Resolve the full path, following every symlink along the way.
        The final component does not need to exist.

    -e, --canonicalize-existing
        Like -f, but every part of the path must actually exist.
        Prints an error if anything is missing.

    -m, --canonicalize-missing
        Like -f, but no part of the path needs to exist.

    -n, --no-newline
        Do not print a newline at the end of the output.

    -q, --quiet
        Do not print error messages when a file is not a symlink.

    -z, --zero
        End each output line with a zero byte instead of a newline.

    --help
        Show a short help message and quit.

EXAMPLES
    readlink mylink
        Print where the symlink mylink points to.

    readlink -f /usr/bin/python
        Print the full real path, resolving all symlinks.
        Example output: /usr/bin/python3.12

    readlink -e /etc/localtime
        Print the real path of the timezone file if it exists.

SEE ALSO
    ln(1), realpath(1)
