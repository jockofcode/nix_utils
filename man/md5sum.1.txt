NAME
    md5sum - calculate or check MD5 fingerprints for files

WHAT IT DOES
    md5sum creates a fingerprint (called a checksum) for each file using
    a formula called MD5. The fingerprint is a 32-character string of
    letters and numbers. If two files have the same fingerprint, they
    contain exactly the same data. If even one byte is different, the
    fingerprints will not match.

    You can also use md5sum to check whether files you downloaded are
    still correct and have not been corrupted.

    Note: MD5 is useful for checking accidental file corruption, but is
    not secure enough for protecting against deliberate tampering. For
    security use sha256sum or sha512sum instead.

HOW TO USE IT
    md5sum [options] [FILE]...

    When no FILE is given, md5sum reads from standard input.

OPTIONS
    -b, --binary
        Read the file in binary mode (marks output with *).

    -c, --check
        Read fingerprints from FILE and check that the files still match.

    --tag
        Print the result in BSD style: MD5 (filename) = fingerprint.

    --quiet
        When checking, only report files that FAILED, not the ones that
        passed.

    --status
        When checking, do not print anything; use the exit code only.

    -w, --warn
        When checking, warn about lines that are not valid fingerprint lines.

    --help
        Show a short help message and quit.

EXAMPLES
    md5sum photo.jpg
        Print the MD5 fingerprint for photo.jpg.

    md5sum *.txt > checksums.txt
        Save the fingerprints of all .txt files.

    md5sum -c checksums.txt
        Check that all files still match their saved fingerprints.

SEE ALSO
    sha256sum(1), sha512sum(1), cksum(1)
