NAME
    od - dump file contents in octal, hex, or other number formats

WHAT IT DOES
    od (octal dump) shows the raw bytes of a file as numbers. By default
    it uses octal (base 8), but you can switch to hexadecimal (base 16),
    decimal (base 10), or display the characters directly.

    This is useful for examining binary files or for seeing exactly what
    bytes a file contains — including invisible characters like null
    bytes, tabs, or control codes.

HOW TO USE IT
    od [options] [FILE]...

    When no FILE is given, od reads from standard input.

OPTIONS
    -A RADIX, --address-radix=RADIX
        Choose the number base for the address column:
          o  octal (default)
          x  hexadecimal
          d  decimal
          n  no address column

    -t TYPE, --format=TYPE
        Choose how to display the bytes. Common types:
          o1  octal bytes (one byte at a time)
          x1  hex bytes
          d1  decimal bytes
          c   printable characters (invisible ones shown as names)
          a   named characters

    -j N, --skip-bytes=N
        Skip the first N bytes before showing anything.

    -N N, --read-bytes=N
        Only show N bytes total.

    -v, --output-duplicates
        Show all lines even if they are identical (no abbreviation).

    -c
        Show bytes as characters; use names like \n for newline, \t
        for tab, and NUL for zero bytes.

    --help
        Show a short help message and quit.

EXAMPLES
    od file.bin
        Show file.bin as octal two-byte integers.

    od -c message.txt
        Show each character by name, revealing hidden control characters.

    od -t x1 photo.jpg | head
        Show the first few lines of a photo as individual hex bytes.

    od -A x -t x1 -v data.bin
        Show bytes in hex with hex addresses.
