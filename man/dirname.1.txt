DIRNAME(1)                       User Commands                       DIRNAME(1)

NAME
    dirname - get the folder part of a file path

WHAT IT DOES
    When you have a full path like /home/alice/documents/report.txt,
    dirname gives you just the folder part: /home/alice/documents.
    It is the opposite of basename, which gives you the file name.

    Think of it like cutting the file name off the end of an address so
    you only keep the street and city part.

    If the path has no folder (just a file name), dirname prints a dot
    (.) to mean "the current folder."

HOW TO USE IT
    dirname NAME...

OPTIONS
    -z, --zero
        End each output line with a zero byte instead of a newline.
        This helps other programs that handle file names with spaces.

    --help
        Show a short help message and quit.

EXAMPLES
    dirname /home/alice/report.txt
        Prints: /home/alice

    dirname /usr/bin/grep
        Prints: /usr/bin

    dirname report.txt
        Prints: .   (meaning the current directory)

    dirname /
        Prints: /
