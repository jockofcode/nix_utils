NAME
    sha1sum - calculate or check SHA-1 fingerprints for files

WHAT IT DOES
    sha1sum creates a 40-character SHA-1 fingerprint (checksum) for each
    file. This fingerprint is like a unique ID for the file's contents —
    if the file changes even by one byte, the fingerprint will be
    completely different.

    Note: SHA-1 is useful for checking accidental file corruption but is
    no longer considered secure enough for protecting against deliberate
    tampering. For security, use sha256sum or sha512sum instead.

HOW TO USE IT
    sha1sum [options] [FILE]...

    When no FILE is given, sha1sum reads from standard input.

OPTIONS
    -b, --binary
        Read the file in binary mode (marks output with *).

    -c, --check
        Read fingerprints from FILE and check that the files still match.

    --tag
        Print results in BSD style: SHA1 (filename) = fingerprint.

    --quiet
        When checking, only report files that FAILED.

    --status
        When checking, do not print anything; just use the exit code.

    -w, --warn
        When checking, warn about badly formatted checksum lines.

    --help
        Show a short help message and quit.

EXAMPLES
    sha1sum archive.tar
        Print the SHA-1 fingerprint for archive.tar.

    sha1sum *.txt > sha1sums.txt
        Save fingerprints for all .txt files.

    sha1sum -c sha1sums.txt
        Check that all files still match their saved fingerprints.

SEE ALSO
    sha256sum(1), sha512sum(1), md5sum(1)
