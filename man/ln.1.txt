NAME
    ln - create a link to a file (a shortcut or second name)

WHAT IT DOES
    ln creates a link — a way to make one file accessible from more than
    one location. There are two kinds of links:

    Hard link: a second name for the exact same file data. Both names
    point to the same content and either one can be used to read or
    change the file. The data stays around as long as at least one name
    exists.

    Symbolic link (symlink): a shortcut that points to another file by
    path. If the original file is moved or deleted, the symlink breaks.

HOW TO USE IT
    ln [options] TARGET LINK_NAME
    ln [options] TARGET... DIRECTORY

OPTIONS
    -s, --symbolic
        Create a symbolic link (shortcut) instead of a hard link.

    -f, --force
        If LINK_NAME already exists, remove it first.

    -i, --interactive
        Ask before overwriting an existing file.

    -n, --no-dereference
        When LINK_NAME is a symlink to a directory, do not follow it.

    -r, --relative
        Create a symbolic link that uses a relative path.

    -v, --verbose
        Print the name of each link as it is created.

    --help
        Show a short help message and quit.

EXAMPLES
    ln -s /usr/bin/python3 python
        Create a symlink named "python" that points to python3.

    ln original.txt backup.txt
        Create a hard link so the file is known by two names.

    ln -sf newfile.txt link.txt
        Change an existing symlink to point to a different file.
