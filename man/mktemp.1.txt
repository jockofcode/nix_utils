NAME
    mktemp - create a temporary file or folder with a unique name

WHAT IT DOES
    mktemp creates a brand-new temporary file with a unique name so that
    two programs running at the same time will not accidentally use the
    same file. It prints the name of the file it created so your script
    can use it.

    Temporary files are meant to be used briefly and then deleted when
    you are done.

HOW TO USE IT
    mktemp [options] [TEMPLATE]

    TEMPLATE is a name that ends with at least three X characters. Those
    Xs are replaced with random letters and numbers. Default template is
    something like /tmp/tmp.XXXXXXXXXX.

OPTIONS
    -d, --directory
        Create a temporary folder instead of a file.

    -u, --dry-run
        Print what the name would be without actually creating anything.

    -p DIR, --tmpdir=DIR
        Create the file inside DIR instead of the default temp folder.

    -t
        Treat the template as a relative name inside the temp folder.

    --suffix=SUFFIX
        Add SUFFIX at the very end of the generated name.

    --help
        Show a short help message and quit.

EXAMPLES
    mktemp
        Create a temp file and print its path, for example:
          /tmp/tmp.aB3xKq9pLm

    mktemp myapp.XXXXXX
        Create a temp file with a name like: myapp.r4tZ8w

    mktemp -d
        Create a temporary directory and print its path.

    TMPFILE=$(mktemp) && echo "data" > "$TMPFILE" && rm "$TMPFILE"
        Create a temp file, use it, then clean it up in a script.
