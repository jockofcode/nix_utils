NAME
    hexdump - show the raw contents of a file in hex and text

WHAT IT DOES
    hexdump shows you the raw bytes inside a file using hexadecimal
    numbers (base 16, using digits 0-9 and letters A-F). Every byte in
    the file gets shown as a two-digit hex number.

    This is useful for inspecting files that are not normal text, like
    images, programs, or data files. You can see the exact bytes the
    computer stores.

    The default view shows hex bytes on the left and printable characters
    on the right (dots replace characters that are not printable).

HOW TO USE IT
    hexdump [options] [FILE]...

    When no FILE is given, hexdump reads from standard input.

OPTIONS
    -C, --canonical
        Show the canonical view: hex bytes on the left side, printable
        ASCII characters on the right side. This is the most readable.

    -b
        Show bytes in octal (base 8) instead of hex.

    -d
        Show bytes as unsigned decimal numbers.

    -o
        Show 2-byte groups in octal.

    -x
        Show 2-byte groups in hex.

    -n N, --length=N
        Only show the first N bytes.

    -s N, --skip=N
        Skip the first N bytes before starting.

    -v
        Show all data even if rows are identical (do not abbreviate
        repeated rows with *).

    --help
        Show a short help message and quit.

EXAMPLES
    hexdump -C photo.jpg | head -5
        Show the first few bytes of a photo in canonical format.

    hexdump -C -n 16 file.bin
        Show just the first 16 bytes.

    hexdump -C myfile | grep "48 65 6c 6c 6f"
        Search for the bytes that spell "Hello" in the file.
